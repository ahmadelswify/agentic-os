# Process Backlog

Turn messy brain dumps into organized, prioritized tasks.

## Before You Start

Check if `GOALS.md` exists. If not, tell the user to run `/setup` first.

## How to Do This

1. Read the backlog file thoroughly
2. Read goals to understand priorities
3. Check existing workspace categories and tasks
4. For each item: create task, connect to project, file as reference, or ask about ambiguous items
5. Clear handled items from the backlog
6. Summarize: "Processed your backlog: X urgent, Y important, Z saved for later."

## Important

- Never show file operations, metadata, or priority codes
- Use natural language for priorities
- Batch your questions
